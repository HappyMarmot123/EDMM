### 1.1 MP1 색상 버그 진단 — **원인 확정 (2026-07-02, 실기기 검증)**

**확인된 사실**: 발생 환경 = Android 삼성 인터넷, 증상 범위 = 앱 전반 포인트 컬러.
**실기기 검증 결과**: 삼성 인터넷 "어두운 모드(웹사이트를 어둡게)"가 켜져 있었고, **"밝게 보기"로 전환하면 색상이 정상 적용됨** → **주 가설 A(강제 다크 모드) 확정.**

**유력 원인 A — 삼성 인터넷 강제 다크 모드 (주 가설)**
- 삼성 인터넷의 "어두운 모드 > 웹사이트를 어둡게" 기능은 페이지가 다크 스킴을 선언하지 않으면 색상을 자체 알고리즘으로 변환한다(밝은 채도색이 어둡고 붉게 왜곡되는 전형적 증상).
- 우리 앱은 이미 다크 테마지만 **`color-scheme` 선언과 `theme-color` meta가 없다** → 삼성 인터넷이 "밝은 페이지"로 오판해 강제 변환.
- "앱 전반 포인트 컬러" 증상과 정확히 부합. 수정 비용 극소(선언 추가), 전 브라우저 무해.

**보조 원인 B — Tailwind v4 모던 컬러 출력 (부 가설)**
- Tailwind v4는 `text-[#ff98a2]/45` 같은 불투명도 수식을 `color-mix(in oklab, …)`로, 기본 팔레트를 `oklch()`로 컴파일한다. 구형 Chromium(<111) 기반 브라우저에서는 해당 선언이 통째로 무시된다.
- 모바일 노출 컴포넌트에 `/수식` 사용 10곳 확인. 선언이 무시되면 상속/기본색으로 폴백 → "색상이 적용 안 된 곳이 많다"와 부합.
- Tailwind v4 공식 지원선은 Chrome 111+/Safari 16.4+. 삼성 인터넷 최신(v23+ ≈ Chromium 115+)은 지원하나 **구버전 잔존 기기**에서 재현 가능.

**수정 전략 (원인 확정 반영)**: **M1(`color-scheme: dark` 선언)이 확정 수정** — 페이지가 다크 스킴을 선언하면 삼성 인터넷이 강제 변환을 건너뛰므로, 사용자가 설정을 바꾸지 않아도 정상 렌더링된다. M2(모던 컬러 폴백)는 **보류로 강등** — 구버전 브라우저 이슈가 실제로 보고될 때 재개.

## 3. 기획설계

### 3.1 M1 — 색상 버그 수정 (원인 확정: 삼성 인터넷 강제 다크 모드)

- `global.css` `:root`에 `color-scheme: dark;` 추가
- 루트 레이아웃(`app/layout.tsx`)에 Next.js viewport/metadata API로 `colorScheme: "dark"` + `themeColor: "#07040a"` 선언
- 효과: 페이지가 다크 스킴을 선언하면 삼성 인터넷이 "웹사이트를 어둡게" 변환을 건너뜀 → **사용자가 브라우저 설정을 바꾸지 않아도** 정상 색상 렌더링

**검증**: 사용자 실기기에서 "어두운 모드(웹사이트를 어둡게)"를 다시 켠 상태로 접속 → #ff98a2가 핑크로 렌더링되면 성공.

### 3.2 M3 — 모바일 seek 바 (터치 드래그)

- 데스크톱 `SeekBar`를 직접 재사용하지 않고 **`useSeekDrag` 공용 훅 추출**(포인터 캡처·클램프·Escape 취소·프리뷰 fraction) 후 모바일 전용 `MSeekBar` 컴포넌트 구성 — UI가 다르므로(치수·라벨 위치·툴팁 없음) 뷰만 분리, 로직은 공유
- 치수 (사용자 피드백 2026-07-02: 현행이 너무 작음 → **기본 사이즈업**): 바 기본 `h-2`(8px) → 드래그 중 `h-3`(12px) 확대, thumb 16px(터치 타깃), 히트 영역 세로 ≥ 32px
- 드래그 중: 채움 바·thumb이 손가락을 따라가고(시각 프리뷰), 하단 현재 시간 라벨이 프리뷰 값으로 갱신·강조. **오디오 seek는 릴리즈 시 1회** (데스크톱과 동일 정책)
- 채움 색: 핑크 `#fd6d94` (미니 바와 통일)
- `mobileFullscreenPlayer`의 기존 클릭 seek(`handleSeek`)을 대체. 닫기 드래그(세로)와 seek 드래그(가로)는 별도 요소라 제스처 충돌 없음 — seek 바에서 `touch-action: none` + 포인터 캡처로 세로 스크롤 개입 차단

### 3.3 M4 — 풀스크린 배경 팔레트 연동 + 아트워크 전환

- `useAlbumColorPalette` + `useArtworkCrossfade`(기존 훅 무수정)를 모바일 풀스크린에 연결
- 배경: 데스크톱 `FullscreenBackdrop`을 재사용하되 모바일 톤(더 어둡게, 하단으로 갈수록 `#050305`)의 오버레이 그라데이션을 위에 얹음 — 기존 정적 그라데이션 대체
- 아트워크: A2와 동일 — top 레이어만 렌더(스냅 아웃) + 280ms 페이드 인, backdrop은 450ms 크로스페이드
- `prefers-reduced-motion` 자동 처리(훅 내장)
- 성능 가드: 저사양 기기 고려해 backdrop blur 강도는 데스크톱보다 낮춤. 문제 시 팔레트 단색 그라데이션으로 강등 가능한 구조

### 3.4 M5+M6 — 컨트롤·미니 바 정비

**모바일 풀스크린 컨트롤**
- shuffle 버튼 **제거** (전역 shuffle 상태는 유지 — 데스크톱에서 켠 상태가 재생에는 계속 반영되며, 모바일에선 조작 UI만 없음)
- play/pause: `hover:bg-[#ffd6e1]` 제거 → `hoverSurface={false}` + `pressFeedback` (탭 시 scale 피드백)
- prev/next: `pressFeedback` 활성화
- 닫기 핸들 바: 현행 유지

**미니 플레이어**
- progress 라인: 흰색 → `#fd6d94`, 높이 1px → 2px 상향 (시인성)
- play/pause 버튼: `pressFeedback` 활성화, 재생 중 아이콘 색 포인트 검토(구체화 단계)
- 배경: `#2b111c` 유지하되 상단 하이라이트 ring 톤 정리 — 큰 변경 없음 (M4가 체감 개선의 중심)

### 3.4.1 M7 — `/search` 뷰포트 높이 overflow

- `global.css`에 전용 클래스 추가:
  ```css
  .app-viewport-height {
    height: 100vh;
    max-height: 100vh;
  }
  @supports (height: 100dvh) {
    .app-viewport-height {
      height: 100dvh;
      max-height: 100dvh;
    }
  }
  ```
- `musicShell/index.tsx:370`의 `h-screen h-[100dvh] max-h-screen max-h-[100dvh]` 4개 클래스를 `.app-viewport-height`로 교체
- 구체화 단계에서 ① 컴파일 CSS에서 기존 페어의 실제 순서 확인(원인 확정) ② 다른 `100dvh` 폴백 페어 사용처(`desktopFullscreenPlayer` 등)도 동일 증상 여부 점검 — 단, 이번 수정 범위는 `/search` 쉘로 한정하고 나머지는 증상 확인 시 확장
- 검증: 모바일 뷰포트(URL 바 표시 상태)에서 세로 스크롤/overflow 없음, 데스크톱 무변화

### 3.5 가드레일

1. **데스크톱 무영향**: 공유 컴포넌트(`PlayerControlButton`, `IconToggleButton`, 훅)의 기본 동작 변경 금지 — 모바일 사용처에서 기존 opt-in prop 활성화만. 새 공용 훅(`useSeekDrag`) 추출 시 데스크톱 `SeekBar`는 동작 동일성을 테스트로 고정
2. **전역 상태 불변**: `audioPlayerProvider` 수정 금지. shuffle 비활성화는 UI 제거로만
3. M1의 `color-scheme`은 전역 선언이지만 앱이 전면 다크 테마이므로 데스크톱에도 무해(오히려 폼 컨트롤 다크 렌더링 정합)
4. 매 Task: `npm test` 전체 + `tsc` + lint 통과, 데스크톱 테스트 회귀 0

### 3.6 검증 계획

- 단위: `MSeekBar` 드래그/취소/클램프, 모바일 풀스크린 아트워크 단독 렌더, shuffle 버튼 부재, 미니 바 클래스
- 수동(사용자, 실기기 삼성 인터넷): ① 핑크 컬러 정상 렌더 ② 풀스크린 배경이 앨범 컬러 연동 ③ seek 드래그 ④ 탭 피드백 ⑤ 데스크톱 뷰 회귀 없음
